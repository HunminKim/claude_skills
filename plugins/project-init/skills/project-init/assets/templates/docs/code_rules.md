# 코드 작성 규칙 (Code Assistant Rules)

> 이 파일이 상세 규칙의 기준이다. CLAUDE.md는 요약만 담는다.

---

## 1. 언어 규칙
- 문서, 새 주석은 한국어로 작성 (기존 영어 주석 유지 가능)
- 영어는 라이브러리명, API명, 고유 명사에만 사용

---

## 2. 작업 프로세스
- 작업 전 요구사항 / 제약조건 / 입출력을 5~10줄로 요약
- 여러 파일 수정 시: 계획 → 승인 → 순차 실행
- 요구사항 외 기능 추가 금지 (필요 시 "추가 제안"으로 분리)
- 불명확한 부분은 반드시 Assumption으로 명시
- 정의부(상수, enum, config, schema)는 반드시 명시적으로 확인 후 인용
- 정의부 확인 없이 코드 수정 금지

---

## 3. 코드 구조
- 과도한 추상화/레이어 금지
- 함수 호출 깊이:
  - 기본: 1~2단계
  - 예외: 테스트/의존성 분리 시 3단계 허용 (사유 명시)
- 조건문 중첩 최소화 (Early Return 사용)
- 반복문 중첩 최대 2단계
- 함수 길이:
  - 권장 50줄 이내
  - 초과 시 분리 검토 결과 1줄 작성
- 함수는 단일 책임 유지
- 함수명은 역할이 명확해야 함

---

## 4. Clean Code
- 명확하고 일관된 네이밍 (축약어 금지)
- 중복 코드 제거 → 공통 함수로 분리
- 매직넘버 금지 → 상수화 + 의미 주석
- 주석은 Why 중심
- 인터페이스 계약은 반드시 명시:
  - 입력/출력
  - shape / dtype
  - 단위 / 범위

---

## 5. 에러 처리 및 로그
- 빈 except 금지
- 에러 메시지는 원인 + 상황 포함
- 로그 레벨 구분:
  - DEBUG: 내부 흐름
  - INFO: 주요 이벤트 / 외부 입력
  - ERROR: 실패
- 민감 정보 출력 금지

---

## 6. 보안 및 의존성
- 외부 입력은 반드시 검증
- 파괴적 작업 기본 비활성화
- 새 라이브러리 추가 시:
  - 도입 이유
  - 대안
  - 선택 근거 명시
- 하드코딩 금지 → config / env 사용
- 파일 경로는 pathlib 사용
- 암묵적 동작 금지 (side effect, implicit conversion)

---

## 7. 인덱스 / 매핑 규칙

**원칙**
- 정의부 먼저 확인 후 코드에 인용
- 코드 순서 ≠ 실제 값
- 인덱스 직접 접근 금지
- Enum + Dict 사용
- Single Source of Truth 유지
- 외부 인덱스 / 내부 인덱스 분리

**권장 패턴 (Python)**
```python
from enum import IntEnum

class CnnOutput(IntEnum):
    BOTTLE = 0
    PHONE = 2

class ActionType(IntEnum):
    DRINKING = 0
    PHONE = 2

CNN_TO_ACTION = {
    CnnOutput.BOTTLE: ActionType.DRINKING,
    CnnOutput.PHONE: ActionType.PHONE,
}
```

**위험 신호**
- 상수 정의가 다른 파일에 있음
- `NAMES[idx]` 접근
- `if idx == 0` 하드코딩
- 외부 인덱스를 그대로 사용

---

## 8. 테스트
- 핵심 로직은 단위 테스트 필수
- 반드시 테스트할 것:
  - 인덱스 매핑
  - 변환 로직
- 경계값 / 예외 포함
- 변경 시 리스크 명시

---

## 9. ML 도메인
- 입력/출력 명시 (shape, dtype, 단위)
- 랜덤 시드 고정
- 전처리 / 모델 / 후처리 분리
- 모델 출력 인덱스 문서화
- 출력 의미 명시 (logits / probability, threshold 기준)

---

## 10. 답변 형식

**코드 요청 시**
1. 결론 / 요약
2. 전체 코드
3. 코드 설명
4. 사용 방법

**단순 질문 시**
- 간결하게 답변

---

## 11. 토큰 관리

**사전 확인 필요**
- 1000줄 이상 파일
- 5MB 이상 파일
- 이미지 / PDF / 데이터셋
- `**/*` 전체 검색

**예외**
- 사용자 명시 요청 시 즉시 진행
- 일반 코드 검색은 허용

---

## 최종 원칙
- 단순하고 명확하게 작성
- 정의부 기반으로 판단
- 매핑은 한 곳에서 관리
- 테스트로 검증
- 코드 순서가 아니라 정의를 신뢰
